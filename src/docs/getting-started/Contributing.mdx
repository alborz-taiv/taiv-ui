# How to Contribute

<br />

### Developing with Taiv UI

To implement changes to this repository, you will need to npm link to your web project so that any changes you make to the toolkit are reflected in your web project.

1. In the toolkit repo, start a new terminal and run `npm run dev`
2. In your project repo (i.e. web app), run `npm link @taiv/ui`
3. Done! Any changes you make should be reflected in your project.

You can also use Storybook as a starting point for your development by running `npm run storybook` in the toolkit repo.

<br />

### Publishing Changes

1. If you created a new component, hook, etc, make sure you added it to the appropriate `index.ts` and created a placeholder documentation page for it.
2. Once you're satisfied, `npm run build` and make a PR to `staging` in the toolkit repo. Chromatic will automatically run and comment with the Storybook URL for your branch.
3. If you made changes to your consuming project, make a PR there and include that updating Taiv UI is a release requirement.
4. Once we're ready to release - we have a GitHub Actions workflow called **Release** that will merge `staging` into `main` and publish a new version to npm. It needs to be manually triggered.
5. When the new version is released, in the consuming project, run `npm i @taiv/ui` to pull the new version of the package
6. Done!
