# Notifications

[View Mantine Docs](https://v6.mantine.dev/others/notifications/)

Integrated into our web apps, we have a hook-based notification system for showing feedback to users. You can call the hook to show a notification with a single line of code.

### 📦 Import
```tsx
// Top-level import
import { useNotifications } from '@taiv/ui/hooks';

// Somewhere in the component
const notifications = useNotifications();
```

### ✍️ Basic Usage

For most use cases, you can just use the `show()` method to show your notification.
```tsx
notifications.show({ variant: 'success', message: 'Your device has been added successfully!' });
```

### ❌ Generic Error

On top of default `show`, we also have `showError()` which is a convenience method that shows a predefined error notification with a support contact message. It doesnt need anything else passed in.

This is useful to toss quickly into try/catch blocks.

<Story of={NotificationStories.GenericError} />
```tsx
try {
    // Your code here
} catch (e) {
    notifications.showError();
    console.log(e.message);
}
```

## <br />

### 🔄 Loading Notifications

Also on top of our regular show, we have `showLoading()` which shows a loading notification and returns an ID that you can then pass to `hide()` when the operation completes. 

<Story of={NotificationStories.AsyncNotifications} />
```tsx
const handleSave = async () => {
  const id = notifications.showLoading({ message: 'Saving...' });

  try {
    await saveData();
    notifications.hide(id);
    notifications.show({ variant: 'success', message: 'Saved!' });
  } catch {
    notifications.hide(id);
    notifications.showError();
  }
};
```

This is the primary pattern for async notifications, and is useful when we want to show something is happening in the background. 

For example, if a page is scrollable, then a button loading state wouldn't be enough because the user could scroll away and forget.
