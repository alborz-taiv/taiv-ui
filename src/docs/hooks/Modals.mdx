# Modals

[View Mantine Docs](https://v6.mantine.dev/others/modals/)

Integrated into our web apps, we have a hook-based modal system for showing info and confirmation modals programmatically. You can call the hooks to show modals and handle their lifecycle with single blocks of code.

In other cases, we might need a more structured modal (i.e. modals with a form or inputs). For those cases we can't use our hook system. See <a href="/?path=/docs/components-info-modals-formmodal--docs">FormModal</a>.
If you need an even more primitive modal to build in, see <a href="/?path=/docs/components-info-modals--docs">Modal</a>.

    ### 📦 Import

    ```tsx
    // Top-level import
    import { useInfoModal } from '@taiv/ui/hooks';

    // Somewhere in the component
    const infoModals = useInfoModal();
    ```

    <br />

    ### ✍️ Basic Usage

    For most use cases, you can just use the `show()` method to show your info modal.

    ```tsx
    infoModals.show({ variant: 'success', title: 'Saved!', message: 'Your changes have been saved.' });
    ```

    ### 📦 Import

    ```tsx
    // Top-level import
    import { useConfirmationModal } from '@taiv/ui/hooks';

    // Somewhere in the component
    const confirmationModals = useConfirmationModal();
    ```

    <br />

    ### ✍️ Basic Usage

    For most use cases, you can just use the `show()` method to show your confirmation modal.

    ```tsx
    confirmationModals.show({
      variant: 'cancel',
      title: 'Delete Device',
      message: 'Are you sure you want to delete this device?',
      onConfirm: () => deleteDevice(id),
    });
    ```

### 📋 Forms & Structured Layouts

For more structured modals — ones with form inputs, custom content, or layouts that go beyond a simple message — you can't use a hook-based approach (yet).
Instead, you can use <a href="/?path=/docs/components-info-modals-formmodal--docs">FormModal</a> and handle its state within your component.
