# Colors

Our color system is split into two categories: **semantic** and **primitive**.


#### Semantic

Semantic tokens are **functional aliases** (primary, error, success, etc.) used for UI elements, actions, and states — anything that would be affected by theme changes. This will be 90% of use cases.

#### Primitive

Primitive colors are the **raw palettes** (blue, red, purple, etc.) used for purely aesthetic purposes like decorative icons or backgrounds — anything we want hardcoded to a specific color.

Semantics map to primitives - in the future, we can create multiple themes outside of our default one by easily just remapping the semantics to different colors.

<br />

### 📦 Import
```tsx
import {primary, red, colors } from '@taiv/ui/constants';

// Use a semantic color
primary[200]  

// Use a primitive color
red[100]  

// Using a unified import
colors.primary[200]
```

## <br />

### 💡 Best Practices

1. **Default to semantic tokens.** Use `primary`, `error`, etc. for anything functional. This keeps the codebase theme-ready.

2. **Primitives are for decoration only.** If the color communicates meaning (success, error, action), use semantic. If it's purely visual (a purple icon, a salmon background), use primitive.

3. **Reference by shade, not raw hex.** Always use `primary[200]` over `#00A6F4`.
