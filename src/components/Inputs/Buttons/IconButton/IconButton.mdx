# IconButton

[View Mantine Docs](https://v6.mantine.dev/core/button/)

The IconButton component is a customizable wrapper around Mantine's Button component that provides consistent styling and behavior across your application. It extends Mantine v6's ButtonProps while adding custom variants and sizing options.

Use IconButton when you need to trigger some sort of logical action via a click through an Icon. See variants below for different use cases.

### üì¶ Import

```tsx
import { IconButton } from '@taiv/ui';
```

### üí° Best Practices

1. **Use semantic variants**: Choose variants that match the action's intent. For example - destructive actions, like deletions, should use a red 'cancel' button.

2. **Manage states**: Always show loading states for async actions, like form submissions. Use disabled to keep buttons from being clicked until a form is valid.

3. **Don't type out defaults**: The default style is primary, and default size is medium. For most buttons, you can ignore these props to simplify code.

4. **Icon placement**: Use Icons as children.

5. **Use subtle for inverted colors**: When you need a less prominent button but still want to convey semantic meaning, use the `subtle` prop.

6. **Add tooltips for clarity**: Icon-only buttons can be ambiguous - use the `tooltip` prop to provide context on hover.
