# SSOButton

A sign-in button for third-party authentication providers. Use the `provider` shorthand for built-in providers (e.g. Google), or pass `providerName` and `providerIcon` for any custom provider.

### 📦 Import

```tsx
import { SSOButton } from '@taiv/ui';
```

### ✍️ Basic Usage

<Canvas of={SSOButtonStories.Default} />
<Controls of={SSOButtonStories.Default} />

---

### 🔐 Built-in Provider

Use the `provider` shorthand to automatically resolve the name and icon.

<Story of={SSOButtonStories.BuiltInProvider} />

```tsx
<SSOButton provider="google" />
```

## <br />

### 🔧 Custom Providers

For providers not built in, pass `providerName` and `providerIcon` directly.

<Story of={SSOButtonStories.CustomProviders} />

```tsx
import { IconBrandApple } from '@tabler/icons-react';

<SSOButton providerName="Apple" providerIcon={<IconBrandApple size={18} />} />
```

### 🏷️ Custom Labels

The default label is `"Continue with {providerName}"`. Override it by passing children.

<Story of={SSOButtonStories.CustomLabels} />

```tsx

```

## <br />

### ↔️ Full Width

Use `fullWidth` for auth forms and modals where buttons should span the container.

<Story of={SSOButtonStories.FullWidth} />

```tsx

```

## <br />

### 🔀 Provider Override

You can use the `provider` shorthand while still overriding `providerIcon` or `providerName`. Explicit props take priority.

<Story of={SSOButtonStories.ProviderOverride} />

```tsx
<SSOButton provider="google" providerIcon={<CustomGoogleIcon />}>
  Custom icon size override
</SSOButton>
```

## <br />

### 💡 Best Practices

1. **Use `provider` shorthand when available**: For built-in providers like Google, this keeps your code clean and consistent.

2. **Use `providerName` / `providerIcon` for custom providers**: For providers not in the built-in map, pass both props explicitly.

3. **Show loading state**: Always indicate a loading state after click while the OAuth flow is initializing.

4. **Consistent sizing**: Use the same `size` for all SSO buttons in a sign-in form.

5. **Full width for auth forms**: Use `fullWidth` when the SSO buttons live inside a narrow auth form or modal.
