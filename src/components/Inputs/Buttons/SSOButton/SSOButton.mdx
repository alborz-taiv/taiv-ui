# SSOButton

A sign-in button for third-party authentication providers. Pass any provider `name` and `icon` — the component handles the outlined styling and generates a default `"Continue with {name}"` label.

### 📦 Import

```tsx
import { SSOButton } from '@taiv/ui';
```

### ✍️ Basic Usage

<Canvas of={SSOButtonStories.Default} />
<Controls of={SSOButtonStories.Default} />

---

### 🔐 Providers

The component works with any provider. Pass a `name` and an `icon` — use any icon library you like.

<Story of={SSOButtonStories.Providers} />

```tsx
import { IconBrandGoogle, IconBrandApple } from '@tabler/icons-react';

<SSOButton name="Google" icon={<IconBrandGoogle size={18} />} />
<SSOButton name="Apple" icon={<IconBrandApple size={18} />} />
```

### 🏷️ Custom Labels

The default label is `"Continue with {name}"`. Override it by passing children.

<Story of={SSOButtonStories.CustomLabels} />

```tsx

  Sign in with Google
</SSOButton>
```

Use `fullWidth` for auth forms and modals where buttons should span the container.

## <br />

### 💡 Best Practices

1. **Show loading state**: Always indicate a loading state after click while the OAuth flow is initializing.

2. **Consistent sizing**: Use the same `size` for all provider buttons in a sign-in form.

3. **Full width for auth forms**: Use `fullWidth` when the provider buttons live inside a narrow auth form or modal.

4. **Use default labels when possible**: `"Continue with {Provider}"` is the recommended phrasing by most providers. Only override the label if you have a specific reason.

5. **Brand-specific colors**: Use the `styles` prop to match official brand colors when required by provider guidelines.
