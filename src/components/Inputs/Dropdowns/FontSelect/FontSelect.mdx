# FontSelect

The FontSelect component is a specialized wrapper around the custom Select component that provides font selection functionality with live font previews. It displays each font option in its actual typeface and applies the selected font to the input field for immediate visual feedback.

Data is automatically retrieved from the FONTS constants, but custom data can also be supplied to it.

### üì¶ Import

```tsx
import { FontSelect } from '@taiv/ui';
```

### üí° Best Practices

1. **Use searchable for large font lists**: Enable the searchable prop when there are many fonts to improve usability.

2. **Provide clear labels**: Always include descriptive labels to help users understand what they should select.

3. **Handle validation**: Use the error prop to display validation messages when font selections are invalid or required.

4. **Use clearable when appropriate**: Enable clearable for optional font selections where users might want to reset their choice.

5. **Consistent sizing**: Choose appropriate sizes based on your form layout and user context.

6. **Accessibility**: Ensure labels are descriptive and consider using description for additional context.

7. **Font preview**: The component automatically shows font previews in the dropdown, making it easy for users to see how fonts will look.

8. **Custom font data**: You can provide custom font data if you need to limit or extend the available font options.

9. **Performance**: For very large font lists, consider implementing virtualization or lazy loading.

10. **Fallback fonts**: Always ensure selected fonts have appropriate fallbacks for better cross-platform compatibility.
