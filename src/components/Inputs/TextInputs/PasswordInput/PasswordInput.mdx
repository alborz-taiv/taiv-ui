# PasswordInput

[View Mantine Docs](https://v6.mantine.dev/core/password-input/)

The PasswordInput component is a customizable wrapper around Mantine's PasswordInput component that provides secure password entry with visibility toggle functionality. It extends Mantine v6's PasswordInputProps while adding custom sizing options and preset styling.

### üì¶ Import


### üí° Best Practices

1. **Provide clear labels**: Always include descriptive labels to help users understand what password is expected.

2. **Use helpful descriptions**: Provide password requirements and guidelines to help users create strong passwords.

3. **Handle validation**: Use the error prop to display clear validation messages for password requirements.

4. **Consider accessibility**: Ensure labels are descriptive and consider using description for password requirements.

5. **Visibility toggle**: The built-in visibility toggle helps users verify their password entry, improving UX.

6. **Controlled visibility**: Use controlled visibility when you need to programmatically manage the show/hide state.

7. **Consistent sizing**: Choose appropriate sizes based on your form layout and user context.

8. **Password confirmation**: Use separate PasswordInput components for password confirmation workflows.

9. **Security considerations**: Always use HTTPS and implement proper password hashing on the backend.

10. **Custom styling**: Use the styles prop to customize appearance while maintaining consistency with your design system.
