# PieChart

[Recharts documentation](https://recharts.org/)

The PieChart component composes Recharts `Pie`, `Cell`, `Tooltip`, optional `Legend`, and optional `Label` center content. Tooltip slice values go through `getChartFormatter`.

**`format`** accepts only:

- A **format key string** — any key of `numberFormats` (`integer`, `decimal`, `currency`, `percentage`, `multiple`). Default **`format`** is **`'percentage'`**.
- PieChart does **not** support Chart’s **`{ format, options? }`** object form for slice tooltips; the large **`centerContent.value`** line uses **`format`** with **0** decimal places when `value` is set.

Use PieChart for part-to-whole proportions. For time-series or categorical metrics where multiple line/area/bar series share the same X categories, use **Chart**.

### 📦 Import

```tsx
import { PieChart } from '@taiv/ui';
```

### ✍️ Basic usage

The **Default** story uses `format="percentage"` and sample slices. Controls expose `data`, `format`, and toggles; for **`centerContent`**, **`innerRadius`**, **`outerRadius`**, and **`paddingAngle`**, use code (see **WithCenterContent**, **DonutGeometry**, **PaddingAngle**).

```tsx
<PieChart data={slices} format="percentage" height="100%" />

<PieChart
  data={slices}
  format="currency"
  height="100%"
  centerContent={{ value: 1200, title: 'Revenue', subtitle: 'Q1' }}
  innerRadius={0}
/>
```

### 🎛️ Donut geometry (`innerRadius`, `outerRadius`, `paddingAngle`)

**`innerRadius`** — Use **0** for a full pie; default **80** with **`outerRadius` 100** yields a donut.

**`outerRadius`** — Outer radius in px (default **100**).

**`paddingAngle`** — Gap between slices in degrees (default **5**).

### 🏷️ Slice labels (`key`)

## <br />

### 💡 Best practices

1. **Stable slice colors** — Pass **`color`** on each **`PieDataPoint`** so legend swatches and **`Cell`** fills stay aligned.

2. **Match `format` to semantics** — Use **`percentage`** for share-of-total, **`currency`** for money, **`integer`** for counts (same guidance as Chart’s Y-axis choice).

3. **Keep `key` human-readable** — Legend and tooltip titles come from **`key`**; avoid opaque IDs in UI-facing charts.

4. **Prefer a small slice count** — Group long tails into “Other” when there are many categories.

5. **Center `value` and zero** — The large center stat uses a truthy check; use **`subtitle`** / **`title`** alone if **`0`** should not show a big number.

6. **Give the chart a height** — When using `height="100%"`, ensure an ancestor sets a definite height (Storybook examples use a fixed `Box` height).
