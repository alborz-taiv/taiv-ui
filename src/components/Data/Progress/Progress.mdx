# Progress

[View Mantine Docs](https://v6.mantine.dev/core/progress/)

The Progress component is a horizontal progress bar built on Mantine's Progress. Pass **`data`** as **`ProgressDataPoint[]`** for multiple segments; a single **`value`** / **`color`** bar also works for loading, completion, and other cases.

Use Progress for loading states, completion percentages, step indicators, and dashboards where **`data`** splits the bar into comparable segments (for example share of totals). Pair bars with labels or nearby text when the percentage alone would be unclear.

### 📦 Import

```tsx
import { Progress } from '@taiv/ui';
```

## <br />

### 💡 Best Practices

1. **Data**: Prefer **`data`** when the UI should show multiple parts (each point is a **`ProgressDataPoint`**). A single **`value`** / **`color`** is enough whenever one continuous fill fits.

2. **Scale and corners**: Use **`scale`** for bar height and **`cornerRadius`** for how rounded the bar is. Both use the same **`xs`**–**`xl`** preset strings.

3. **Width and layout**: Let the **parent** set width when you can—**`100%`** in a flex or grid column often behaves best. Use **`width`** in **rem** or **`%`** when the layout needs a fixed bar width.

4. **Labels and context**: Add a **`label`** or nearby text when the bar alone would be ambiguous (percent, step, or status). With **`data`**, use **`label`** and **`tooltip`** on each **`ProgressDataPoint`** so segments stay readable.

5. **Colors**: Try to use **Taiv palette values** from **`constants/colors`** for **`color`** and each point’s **`color`**. In multi-segment bars, pick values that stay visually distinct next to one another. For a simple single fill, omitting **`color`** and **`data`** yields **primary 200**.
