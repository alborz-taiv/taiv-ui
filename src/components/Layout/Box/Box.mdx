# Box

[View Mantine Docs](https://v6.mantine.dev/core/box/)

Box is an empty `<div>` that can be used to wrap elements and apply styling.

Before reaching for it, ask whether [Stack](?path=/docs/components-layout-stack--docs), [Group](?path=/docs/components-layout-group--docs),
[Center](?path=/docs/components-layout-center--docs), [UnstyledButton](?path=/docs/components-inputs-buttons-unstyledbutton--docs), or [Card](?path=/docs/components-layout-card--docs)
are better suited for what you're building.

If nothing else fits, Box is essentially the replacement for a generic `<div>`.


### 📦 Import
```tsx
import { Box } from '@taiv/ui';
```

### 📁 Box as a Grouping Element

Generally, there are 2 cases where Box is useful. The first is when you need to group elements together within another layout component - Box is preferred over using `<div>`. 

```tsx
// This stack is comprised of 1 stack (header) and 1 box (content). This is preferred over using a fragment or a div.
<Stack>
  <Stack gap="0">
    <Title variant="header">Header</Title>
    <Title variant="subheader">Subheader</Title>
  </Stack>
  <Box>
    {children}
  </Box>
</Stack>
```

---

### 🎨 Box as a Styled Container

The second use case for Box is for one-off "container" type things. `sx` gives you direct CSS access to style a Box.

<Story of={BoxStories.BoxStyling} />
```tsx
<Box
  sx={{
    backgroundColor: neutral[50],
    padding: '24px',
    borderRadius: '8px',
    border: `1px dashed ${neutral[100]}`,
  }}
>
  <Text weight="semibold">This is a box that has been styled</Text>
</Box>
```

Again, you should only need this sparingly, for example in creating new components. If you are using Box directly on a page, it might be a sign that we either have a more relevant component somewhere or that we need to create a new one.
